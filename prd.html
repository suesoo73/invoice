Invoice / Receipt OCR Service PRD

기업 고객이 invoice 또는 receipt 파일을 업로드하면 OCR로 텍스트와 품목 정보를 추출하고, 검수 후 검색 및 보고서로 활용할 수 있도록 하는 별도 웹 애플리케이션의 1차 제품 요구사항 문서.

1. 제품 목표

핵심 가치 기업 고객이 구매 문서를 업로드하고 OCR 결과를 검수한 뒤 빠르게 검색하고 집계할 수 있게 한다.
1차 우선순위 업로드, OCR 추출, 검색
운영 원칙 회사별 데이터 완전 분리, 비동기 처리, 감사 로그 기록, 휴지통 기반 삭제

2. 범위

포함

  • 별도 웹 앱 제공, 워드프레스 JWT 로그인 연동
  • jpg, png, pdf 다중 업로드
  • 문서 종류를 사용자가 invoice 또는 receipt로 직접 선택
  • OCR 원문 텍스트와 구조화 데이터 저장
  • 품목 행 단위 추출 및 사용자 검수
  • 날짜, 거래처, 금액, 품목명 중심 검색
  • 월간/분기별 단순 집계형 엑셀/PDF 보고서

초기 제외

  • 다중 통화 변환 기능
  • 권한 등급 분리 기능
  • 워드프레스 내부 플러그인 UI

3. 사용자와 운영 정책

고객 구조 한 회사에 여러 사용자가 함께 사용한다.
테넌트 분리 회사별 데이터는 완전히 분리한다.
회사 생성 운영자가 회사 계정을 생성한다.
권한 정책 1차에서는 별도 권한 레벨 없이 동일 기능 사용.
보관 정책 고객이 삭제하기 전까지 보관한다.
삭제 정책 삭제 시 휴지통으로 이동 후 7일 보관한다.

4. 주요 업무 흐름

로그인 → 문서 종류 선택 → 다중 파일 업로드 → 미들웨어 저장 → OCR 작업 큐 등록 → LLM 서버 OCR 추출 → DB에 임시 저장 → 사용자 검수/수정 → 완료 확정 → 검색 및 보고서 반영
  • 웹서버는 LLM 서버에 직접 접근하지 않는다.
  • 모든 OCR 요청은 미들웨어를 통해 비동기 처리한다.
  • OCR 실패 시 자동 재시도는 2회 수행한다.
  • 사용자는 상세 화면에서 수동 재처리를 실행할 수 있다.

5. 기능 요구사항

업로드

  • 사용자는 여러 개의 파일을 한 번에 업로드할 수 있다.
  • 지원 형식은 jpg, png, pdf이다.
  • PDF는 문서 단위로 저장하며 페이지별 개별 저장은 하지 않는다.

OCR 추출

  • 필수 추출 항목: 거래처명, 발행일, 공급가액, 세액, 합계금액, 통화, 품목명, 수량, 단가
  • 선택 추출 항목: 결제수단, invoice 번호, receipt 번호
  • 품목 라인은 모두 추출 대상으로 한다.
  • OCR 원문 전체 텍스트를 함께 저장한다.

검수

  • 사용자는 OCR 결과를 수정할 수 있다.
  • 문서 상태는 임시와 완료를 포함한다.
  • 완료 확정된 데이터가 검색과 보고서의 기준 데이터가 된다.

검색/조회

  • 날짜, 거래처, 금액, 품목명 등으로 검색한다.
  • 문서 원본 파일과 OCR 결과를 함께 볼 수 있어야 한다.

보고서

  • 발행일 기준으로 월간/분기별 집계를 제공한다.
  • 비용 집계, 거래처별 통계, 품목별 통계를 제공한다.
  • 엑셀과 PDF 다운로드를 제공한다.
  • 1차 버전은 단순 집계형 포맷으로 시작한다.

8. 상태 정의

  • uploaded: 파일 업로드 완료
  • processing: OCR 처리 중
  • review: OCR 완료, 사용자 검수 필요
  • completed: 사용자 확정 완료
  • failed: OCR 실패, 자동 재시도 종료
  • deleted: 휴지통 이동 상태

9. API 초안

영역 메서드 경로 설명
인증 POST /api/auth/jwt-login 워드프레스 JWT 검증 후 앱 세션 생성
문서 POST /api/documents 다중 파일 업로드 및 OCR 작업 등록

문서 POST /api/documents/{id}/retry OCR 수동 재처리

10. 시스템 구성

사용자 브라우저 → 별도 웹 앱 → 미들웨어 → LLM 서버(OCR)
사용자 브라우저 → 별도 웹 앱 → DB / 파일 저장소 / 보고서 생성
워드프레스 → JWT 발급 → 별도 웹 앱 인증 연동
  • 웹 앱은 고객 UI와 검색/검수/보고서 기능을 담당한다.
  • 미들웨어는 파일 저장, 작업 큐, 재시도, 내부 인증, 로그 감사, LLM 서버 접근 제어를 담당한다.
  • LLM 서버는 OCR과 구조화 추출만 담당한다.
  • 웹서버와 LLM 서버의 직접 통신은 차단한다.

11. 비기능 요구사항

  • 회사별 데이터 접근은 반드시 테넌트 기준으로 제한한다.
  • 업로드, 수정, 삭제, 재처리, 로그인 이벤트를 감사 로그로 남긴다.
  • 초기 통화는 KRW만 지원한다.
  • 휴지통 데이터는 7일 후 자동 영구 삭제 대상이 된다.
  • OCR 자동 재시도는 최대 2회까지만 수행한다.

12. 1차 개발 우선순위

  1. JWT 로그인 연동 및 회사별 데이터 분리
  2. 다중 파일 업로드와 미들웨어 저장
  3. OCR 작업 큐와 LLM 연동
  4. 문서 목록/검색
  5. 문서 상세/검수/완료 처리
  6. 단순 집계형 보고서와 엑셀/PDF 다운로드
  7. 감사 로그, 휴지통, 자동 정리 배치
